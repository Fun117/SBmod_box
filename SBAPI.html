初めに

SBAPIとは、ビル経営ゲームでMODを作成する時に楽を出来る前提modである。

このページでは開発者向けにSBAPIの使い方を解説していく。

導入方法

SBAPIをmod一覧等からダウンロードしてください。

SBAPISBAPIplusが大体のSBAPI式modの動作環境で必要なものです。

"SBAPI clientside"はSBAPI式modのコードを書いていくspriteです。

SBAPIの基本的な使い方

↓のように使用する。これを使う事で建築リストを階段だけにするとかも出来るぞ！

返り値は[list]modAPIの1番目に返ってきます。

メソッドとは

SBAPIにはdatagetpostの3つメソッドが有る。

基本的にはgetとpostを使って、変数とかめんどいときにdataに押し付ける感じ。

メソッドの紹介

data:ビル経に初期搭載されてるcache機能と独立したcache機能が使えます。

get:ビル経の各プロパティを取得します。

post:ビル経の各プロパティの変更や、猫の召喚等を行います。

メソッド - data

cache機能がメイン。SBAPIplusでは文字列操作が追加される

cache

key,valueの形でcacheする

[SBAPI request (data) (cache)(key)(value)]

get

keyからcacheを取得する

[SBAPI request (data) (get)(key)()]

delete

keyのcacheを削除する

[SBAPI request (data) (delete)(key)()]

EarlyMod-初期搭載modについて

ちょっと作るのが怠いなぁという機能をSBAPIに丸投げ出来る機能です。

維持費無効化

維持費を無効化する。副作用として時間概念が消える

[SBAPI request (post) (EarlyMod)(keepcost)(true)]
